## 工具 1：📝 标题生成器 + Hook 分析（Title & Hook Analyzer）

### 设计目标

标题和开头是自媒体文章最关键的部分——它们决定了读者是否会点进来。CN 需要帮助创作者**写出无法忽略的标题和开头**。

### 标题生成器

```tsx
interface TitleGenerator {
  // 输入
  articleContent: string;          // 文章全文或摘要
  targetPlatform: Platform;        // 目标平台
  targetAudience: string;          // 目标受众描述
  tone: 'informative' | 'provocative' | 'emotional' | 'humorous' | 'authoritative';
  
  // 输出
  generate(): TitleCandidate[];
}

interface TitleCandidate {
  title: string;
  
  // 评分维度
  scores: {
    curiosityGap: number;          // 好奇心缺口 0-10（让人想点进来）
    clarity: number;               // 清晰度 0-10（读者知道会看到什么）
    emotionalPull: number;         // 情感吸引力 0-10
    seoScore: number;              // SEO 友好度 0-10
    platformFit: number;           // 平台适配度 0-10
  };
  
  overallScore: number;            // 综合评分
  
  // 分析
  strengths: string[];             // 优点
  weaknesses: string[];            // 缺点
  improvement: string;             // 改进建议
}
```

### Hook 分析（开头段分析）

```tsx
interface HookAnalysis {
  // 开头段落评估
  hookType: 
    | 'question'            // 提问式："你有没有想过……"
    | 'statistic'           // 数据式："据统计，87% 的人……"
    | 'story'               // 故事式："上周三，我收到一封邮件……"
    | 'provocative'         // 挑战式："你以为你了解 AI，但其实不然。"
    | 'pain_point'          // 痛点式："你是不是也遇到过这种情况……"
    | 'promise'             // 承诺式："读完这篇文章，你会学到……"
    | 'contrast'            // 反差式："别人用 AI 写 10 万字/天，我却……"
    | 'none';               // 没有明显的 Hook
  
  // 评分
  hookStrength: number;             // Hook 强度 0-10
  readOnProbability: number;        // 预估读者继续阅读的概率
  
  // 问题
  issues: string[];                 // "开头过长""缺少与读者的直接关联""信息量过大"
  
  // 建议
  alternativeHooks: string[];       // AI 生成的替代开头
}
```

## 工具 2：📊 可读性评分面板（Readability Panel）

### 设计目标

实时分析文章的**可读性指标**，确保内容对目标受众友好。

### 可读性评分体系

```tsx
interface ReadabilityScore {
  // 核心指标
  overallScore: number;            // 综合可读性 0-100
  gradeLevel: string;              // 适合的阅读水平
  
  // 细分维度
  dimensions: {
    sentenceLength: {
      avg: number;                 // 平均句长
      tooLong: number;             // 过长句子数量（>40字）
      score: number;               // 评分 0-10
    };
    paragraphLength: {
      avg: number;                 // 平均段落长度
      tooLong: number;             // 过长段落数量
      score: number;
    };
    vocabularyComplexity: {
      technicalTerms: string[];    // 专业术语列表
      rareWords: string[];         // 生僻词列表
      score: number;
    };
    structureClarity: {
      hasHeadings: boolean;        // 是否有小标题
      hasSummary: boolean;         // 是否有总结段
      logicalFlow: number;         // 逻辑流畅度 0-10
      score: number;
    };
    scanability: {
      boldKeywords: number;        // 加粗关键词数量
      bulletLists: number;         // 列表数量
      visualBreaks: number;        // 视觉分隔数量
      score: number;               // 可扫读性评分
    };
  };
  
  // 针对不同受众的建议
  audienceAdaptation: {
    general: string;               // 通用读者建议
    professional: string;          // 专业读者建议
    casual: string;                // 休闲读者建议
  };
}
```

## 工具 4：🔍 SEO 优化建议器（SEO Advisor）

### 核心功能

```tsx
interface SEOAnalysis {
  // 关键词分析
  keywords: {
    primary: string;               // 主关键词
    secondary: string[];           // 次要关键词
    density: Map<string, number>;  // 各关键词密度
    naturalness: number;           // 关键词融入的自然度 0-10
  };
  
  // 标题 SEO
  titleSEO: {
    containsKeyword: boolean;
    length: number;                // 标题长度
    isOptimalLength: boolean;      // 是否在最佳长度范围
  };
  
  // 结构 SEO
  structureSEO: {
    hasH2Tags: boolean;            // 是否有子标题
    keywordInHeadings: boolean;    // 子标题是否包含关键词
    hasMetaDescription: boolean;   // 是否有摘要/描述
    internalLinks: number;         // 内链数量
    externalLinks: number;         // 外链数量
  };
  
  // 竞争分析
  competitiveInsight: {
    suggestedTopics: string[];     // 建议补充的话题点
    differentiator: string;        // 与同类内容的差异化建议
  };
}

## 工具 5：💡 观点提取与论证强化（Argument Enhancer）

### 设计目标

帮助创作者**提炼核心论点**，强化论证逻辑，让文章更有说服力。

### 功能

- **论点提取**：AI 从文章中自动识别核心论点和支撑论据
- **论证结构分析**：评估论证链是否完整（论点 → 论据 → 论证 → 结论）
- **论据建议**：为薄弱的论点建议更有力的支撑（数据、案例、引用）
- **反驳预演**：AI 模拟读者可能的反驳，帮助创作者提前回应
- **MECE 检查**：检查论述是否"相互独立、完全穷尽"

### 论证分析 UI

```

│  ✅ 有具体案例（CN 的分层上下文系统）
│  ✅ 有数据支撑（效率提升数据）

## 工具 6：🔥 热点关联引擎（Trend Connector）

### 设计目标

帮助创作者将自己的内容与**当前热点话题**关联，增加传播力。

### 核心功能

- **热点监测**：接入趋势数据源（微博热搜、知乎热榜、百度指数等）
- **关联建议**：AI 分析创作者的内容领域，推荐可关联的热点
- **切入角度建议**：为每个热点建议具体的内容切入角度
- **时效性提醒**：标注热点的时效窗口（"这个话题预计还有 2 天热度"）

## 工具 7：📅 内容日历（Content Calendar）

### 设计目标

帮助自媒体创作者**规划和管理内容发布节奏**。

### 核心功能

- **日历视图**：月度/周度视图展示计划中和已发布的内容
- **内容管道**：想法 → 大纲 → 草稿 → 修改 → 待发布 → 已发布
- **系列管理**：管理系列文章（上/中/下）或专栏
- **发布提醒**：在计划发布日期前提醒
- **频率分析**："你上周发了 3 篇，本周还没发"

| **技能名称** | **功能** | **触发方式** |
| --- | --- | --- |
| 改写优化 | 让段落更简洁有力、更适合阅读 | 默认技能，快捷键 Tab |

| 平台适配 | 将内容转换为特定平台的风格 | 导出面板 |

## 自媒体模式的专属审查维度

1. **可读性审查**：句长、段长、词汇复杂度是否适合目标受众
2. **标题吸引力审查**：标题的好奇心缺口、情感力度、清晰度评分
3. **Hook 强度审查**：开头段落是否在前 3 句抓住读者
4. **论证逻辑审查**：论点-论据链是否完整和有说服力
5. **SEO 审查**：关键词密度、标题优化、结构优化
6. **平台适配审查**：字数、格式、排版是否符合目标平台规范
7. **CTA 审查**：文章是否有明确的行动号召
