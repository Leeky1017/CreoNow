### 这是什么？

AI 在不同的对话/会话中生成代码时，经常会**重新发明轮子**——明明项目里已经有一个做某件事的函数，它又写了一个功能几乎一样但名字不同的新函数。

打个比方：你让三个人分别帮你整理厨房，结果每个人都买了一套新的收纳盒，而不是用你已有的。最终你有三套功能重叠的收纳系统。

### 🔍 具体表现

- **同功能多实现**：项目里有 `formatDate()`、`convertDate()`、`parseDateString()` 三个函数，做的事情大同小异
- **相似的错误处理块**：到处都有 try-catch，每个地方写法略有不同，但逻辑本质一样
- **重复的类型定义**：同一个数据结构在不同文件里被定义了多次，字段名可能还有细微差异
- **复制-变异模式**：AI 复制了一段已有代码，做了微小修改，而不是复用或重构原有代码

### ⚠️ 为什么有害？

1. **改一处漏十处**：修复一个 bug 时只改了一个版本，其他副本仍然有问题
2. **行为不一致**：用户在不同场景下遇到同一个功能，但表现略有不同（因为底层是不同的实现）
3. **体积膨胀**：重复代码直接增加文件大小和项目复杂度

### ✅ Owner Review 要点

- [ ]  让 Agent 在写新函数前**先搜索项目中已有的类似功能**
- [ ]  定期做「去重审计」：搜索功能相似的函数并合并
- [ ]  在创建新函数/组件之前，必须先搜索项目中已有的相似实现，优先复用
- [ ]  建立一个「核心工具函数清单」，让 Agent 知道哪些功能已经有了
