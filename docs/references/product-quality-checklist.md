# 产品质量检查单

> **前置阅读**：`AGENTS.md`（必读）→ 本文档
> **用途**：产品完整性与用户体验的质量门禁。
> **何时阅读**：实现新功能、提交 PR 前、执行审计时。

---

## 一、产品完整性铁律

### 禁止 Ghost UI

| 规则 | 说明 |
|------|------|
| UI 入口必须有后端实现 | 菜单项/按钮存在 → 对应 Service 必须已实现 |
| Stub Service ≠ 可发布 | 如果 Service 返回 mock 数据，UI 侧不得展示为可用功能 |
| 检测方式 | `scripts/service-stub-detector-gate.ts` 自动扫描 |

### 禁止功能降级静默化

| 规则 | 说明 |
|------|------|
| 功能不可用必须告知用户 | 不能静默降级让用户以为正常 |
| 降级有明确视觉提示 | Toast / Banner / 禁用态 |
| 恢复后自动回到正常状态 | 不需要用户手动刷新 |

## 二、错误 UX 规范

### 错误消息层级

| 层级 | 谁看 | 内容 |
|------|------|------|
| 用户错误消息 | 终端用户 | 友好的自然语言描述 + 建议操作 |
| 开发者日志 | 开发者 | 错误码 + 堆栈 + 上下文参数 |
| 监控指标 | 运维 | 错误频率 + 影响范围 |

### 错误消息规则

| 规则 | 正确示例 | 错误示例 |
|------|---------|---------|
| 不暴露错误码给用户 | 「保存失败，请稍后重试」 | 「Error: DOC_SAVE_FAILED」 |
| 不暴露技术栈 | 「文件格式不支持」 | 「TypeError: cannot read property...」 |
| 提供可操作建议 | 「网络不可用，请检查连接后重试」 | 「网络错误」 |
| 使用 `getUserFacingErrorMessage()` | 统一入口转换错误码→消息 | 组件内硬编码错误文案 |

### 常见错误 UX 反模式

| 反模式 | 危害 | 修复 |
|--------|------|------|
| `window.confirm()` / `window.alert()` | 原生弹窗，风格不统一 | 使用自定义 Dialog 组件 |
| 直接渲染 `error.code` | 用户看到 `EDOC_NOT_FOUND` | 走 `errorMessages.ts` 映射 |
| `catch { }` 空处理 | 用户无反馈 | 至少显示 Toast |
| 错误消息裸英文 | 未 i18n | 使用 `t('error.xxx')` |

## 三、视觉质量 9 维度检查

| 维度 | 检查要点 | 不合格标准 |
|------|---------|-----------|
| **Design Token** | 所有色值/间距走 Token | 出现硬编码 hex/rgba |
| **Primitive 复用** | features/ 中使用 Primitive 组件 | 使用原生 `<button>/<input>` |
| **i18n** | 用户可见文案走 `t()` | 裸字符串字面量 |
| **组件复用** | 检查是否有可复用的已有组件 | 重复实现已有功能 |
| **CSS 规范** | 使用 Tailwind + Token 类 | 内联 style、BEM 类名混用 |
| **无障碍** | ARIA 属性完整 | 图标按钮无 `aria-label` |
| **响应式** | 关键断点适配 | 固定宽度硬编码 |
| **品牌一致性** | 视觉 DNA 对齐（冷灰/紧凑/克制） | 风格跳脱（圆胶囊/强饱和色/重阴影） |
| **动效** | 过渡使用 Token | 无过渡或硬编码 duration |

## 四、用户旅程关键断点

### 核心路径（必须可走通）

| # | 路径 | 验证要点 |
|---|------|---------|
| 1 | 启动 → 首屏 | < 3s 白屏时间 |
| 2 | 新建文档 → 编辑 → 保存 | 自动保存 + 手动保存均正常 |
| 3 | AI 辅助写作 | 调用→等待→结果插入 全链路 |
| 4 | 导出 | 至少支持 Markdown 导出 |
| 5 | 设置修改 | 修改→保存→生效→重启后保持 |

### 各断点的错误兜底

| 断点 | 必须有的兜底 |
|------|-------------|
| 启动失败 | 友好错误页面（非白屏/崩溃） |
| 保存失败 | Toast 提示 + 数据不丢失（内存保留） |
| AI 调用失败 | 超时/限流/错误 三种情况有不同提示 |
| 导出失败 | 文件写入失败时提示原因（权限/空间） |

## 五、前端静态代码审计重点

### P0 问题（必须修复）

| 问题 | 检测方式 | 修复标准 |
|------|---------|---------|
| 裸字符串（未 i18n） | grep 用户可见文案 | 100% 走 `t()` |
| 原生 HTML 元素（绕过 Primitive） | ESLint `no-native-html-element` | features/ 中 0 个原生元素 |
| 硬编码颜色 | ESLint `no-raw-tailwind-tokens` | 0 个硬编码色值 |
| 缺少错误边界 | `error-boundary-coverage-gate.ts` | 关键路由有 ErrorBoundary |

### P1 问题（应该修复）

| 问题 | 检测方式 |
|------|---------|
| 缺少 ARIA 属性 | 手动审查交互元素 |
| 缺少 loading 状态 | 异步操作必须有 loading 反馈 |
| 缺少 empty 状态 | 列表/搜索结果为空时的展示 |

### P2 问题（建议修复）

| 问题 | 检测方式 |
|------|---------|
| 动效不一致 | 检查 transition 是否统一使用 Token |
| 间距不规整 | 4px 网格对齐 |
| 可拖放区域缺少视觉提示 | 拖拽操作有 hover + drop zone 反馈 |

## 六、PR 提交前产品质量自检

```
[ ] 新增 UI 入口是否有对应后端实现？（无 Ghost UI）
[ ] 错误路径是否有用户友好消息？（无裸错误码）
[ ] 用户可见文案是否走 i18n？（无裸字符串）
[ ] 关键用户路径是否可走通？（启动→编辑→保存→AI→导出）
[ ] 是否使用 Primitive 组件？（无原生 HTML 元素）
[ ] 颜色/间距/圆角是否走 Token？（无硬编码值）
[ ] 交互状态是否完整？（hover/focus/active/disabled）
[ ] Dark/Light 主题是否正常？
[ ] 新组件是否有 Storybook Story？
```
