# 99.5 明确不做的事情（Non-Goals）

> CreoNow 前端的设计边界——以下是我们**明确不做**的事情，以避免范围蔓延和技术选型混乱。

| # | Non-Goal | 理由 / 约束来源 |
|---|----------|-----------------|
| 1 | **不提供浅色主题** | Dark-only 设计，见 [20.8 响应式规范](../20-design-v3/20.8-responsive.md)。深色模式是 Cinematic Calm 美学的基石 |
| 2 | **不使用 CSS-in-JS** | 使用 Tailwind CSS + Design Token CSS Variables，不引入 styled-components / Emotion / CSS Modules |
| 3 | **不使用 Redux / MobX** | 状态管理统一使用 Zustand 5，见 [10.4 状态管理](../10-architecture/10.4-state-management.md) |
| 4 | **不使用 Next.js / SSR** | CreoNow 是纯 CSR 的 Electron 桌面应用，不需要服务端渲染 |
| 5 | **不使用 contextBridge 以外的 IPC 方式** | Electron `contextBridge` 是渲染进程与主进程通信的唯一合法通道，见 [10.3 IPC 通道](../10-architecture/10.3-ipc-channels.md) |
| 6 | **不从 V1/V2 复制代码** | 只继承设计理念与视觉方向，代码从零实现。V1/V2 源码仅作参考，见 [30-reference-v1-v2/](../30-reference-v1-v2/) |
| 7 | **不支持移动端 / Web 端** | CreoNow 是桌面 Electron 专属应用，不做响应式 Web 或移动端适配 |
| 8 | **不构建自定义图标库** | 统一使用 Lucide React（1.5px stroke），见 [20.10 图标系统](../20-design-v3/20.10-icons.md) |
