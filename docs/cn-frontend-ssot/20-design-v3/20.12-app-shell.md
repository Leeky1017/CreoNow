## 一、Shell 层级结构（§23.1）

```
<AppShell>                          z-index 层级
  ├── <IconRail/>                   z: 20
  ├── <ContextPanel/>               z: 20
  ├── <MainContent>
  │     └── <RouterOutlet/>         z: 0（编辑器内容基准层）
  ├── <RightSidebar/>               z: 20
  ├── <AIFab/>                      z: 100（--z-fab）
  ├── <CommandPalette/>             z: 200（--z-command）
  ├── <SettingsModal/>              z: 300（--z-modal）
  └── <ToastProvider/>              z: 400（--z-toast，最高层）
```

### Shell 布局参数

| **元素** | **默认尺寸** | **Zen Mode** | **Focus Mode** | **多栏模式** |
| --- | --- | --- | --- | --- |
| Icon Rail | 48px 固定宽 | 隐藏 | 隐藏 | 隐藏 |
| Context Panel | 240px，可折叠为 0 | 隐藏 | 隐藏 | 隐藏 |
| Main Content | `flex-1`（剩余宽度） | 100vw | 50vw（可拖调） | 按分栏比例 |
| Right Sidebar | 320px，可关闭 | 隐藏 | 50vw（AI 面板） | 不显示（AI 面板嵌入分栏） |
| AI FAB | 44×44px，右下角 | 隐藏 | 隐藏 | 隐藏（AI 面板已在分栏中） |

### Shell 状态机

| **状态** | **触发** | **互斥** | **存储位置** |
| --- | --- | --- | --- |
| Normal（默认） | 应用启动 / 退出其他模式 | — | `useAppStore.mode = 'normal'` |
| Zen Mode | `Alt+Z` / 工具栏图标 | 与 Focus Mode 互斥 | `useAppStore.mode = 'zen'` |
| Focus Mode | FAB 长按 500ms / `Ctrl+Shift+F` | 与 Zen Mode 互斥 | `useAppStore.mode = 'focus'` |
| Multi-Pane | `Ctrl+\` / Context Panel 右键 | 与 Zen/Focus 互斥（退出后恢复） | `useAppStore.panes[]` |

**模式切换动画时长统一为 `--duration-slow`（300ms）**，各面板用 `translateX` + `opacity` 组合过渡。

### 全局快捷键注册

Shell 层在 `useEffect` 中注册全局 `keydown` 监听，优先级高于编辑器内部快捷键：

- `Ctrl+K` → 打开 CommandPalette
- `Alt+Z` → 切换 Zen Mode（仅在 `/editor/*` 路由）
- `Ctrl+Shift+F` → 切换 Focus Mode（仅在 `/editor/*` 路由）
- `Ctrl+,` → 打开 SettingsModal
- `Ctrl+B` → 折叠/展开 Context Panel
- `Escape` → 优先关闭最高层浮层（Toast 不响应，Modal → CommandPalette → Focus/Zen）

## 二、标题栏（§29.1）

**方案：自定义极简标题栏，`frame: false` + 手动绘制。** 高度 32px。

```
┌─────────────────────────────────────────────────────────┐
│ ✦  │ [当前文档名 ▾]                    │ — □ ✕ │  ← 32px
└─────────────────────────────────────────────────────────┘
  ↑       ↑                                  ↑
图标区   Tab 触发区（点击弹出文档列表）    系统按钮区
(24px)                                   (Notion 同款)
```

| **区域** | **宽度** | **内容** | **行为** |
| --- | --- | --- | --- |
| **图标区**（左） | 48px（与 Icon Rail 等宽） | ✦ 应用图标 SVG，20×20px，居中 | 不可点击，整条标题栏为拖拽区（`-webkit-app-region: drag`） |
| **Tab 触发区**（中） | flex-1 | 当前激活文档标题 + `▾` chevron；未保存时标题前有 `•` | 点击弹出文档切换列表；`-webkit-app-region: no-drag` |
| **系统按钮区**（右） | 约 104px | 最小化 `—`、最大化 `□`、关闭 `✕`（Notion 同款） | `-webkit-app-region: no-drag`；关闭按钮 hover 变红（`#EF4444`） |

- **背景色**：`#000000`（与 Icon Rail 齐平）
- **标题文字**：Inter 13px，`#737373`
- **Zen Mode 中**：标题栏保留但高度缩为 24px，仅保留系统按钮区

## 三、AI 三模式架构（§29.2）

### AI 模式一：AI 侧边栏（Sidebar Mode）

- AI 面板默认宽度 320px，Docked（推挤编辑区）
- 通过 Icon Rail 底部 AI 图标或 FAB 双击开启
- 编辑器与 AI 面板同时可见，互相联动（`@` 引用当前选中内容）

### AI 模式二：AI Focus 模式（Focus Mode）

- **编辑器完全隐藏**，AI 面板占满 Icon Rail 右侧全部空间
- 对话区更宽（最大宽度 720px 居中）
- 触发：标题栏 `✦ Focus` 图标按钮，或 `Alt+A`（macOS `Opt+A`）
- 状态独立：Focus 模式下的对话历史与 Sidebar 模式共享

### AI 模式三：Inline 工具栏（选中即弹出）

**工具栏布局（两行，参考 Notion）：**

```
┌───────────────────────────────────────────────┐
│  T  A  B  I  U  ~  </>  Link  ···  Comment  |  AI  │  ← 第一行
│───────────────────────────────────────────────│
│ Skills: 提升写作  校对  解释  改写语气  重格式化  ⋯ │  ← 第二行
└───────────────────────────────────────────────┘
```

**视觉规范：** 宽度 320–520px，背景 `#161616`，圆角 8px，阴影 `0 8px 24px rgba(0,0,0,0.6)`，`z-index: --z-overlay`（200）。

### FAB 球（全局 AI 入口——始终可见）

> **Note**: FAB 完整交互规格以 [20.2 组件规范 §5](20.2-component-specs.md) 为准。

| **入口** | **触发** | **结果** |
| --- | --- | --- |
| FAB 球 | 右下角始终可见 | 单击→Sidebar / 双击→快速菜单 / 长按→Focus / 有选中文字时→Inline |
| Icon Rail AI 图标 | 点击 Bot 图标 | 打开/关闭 AI Sidebar |
| 标题栏 Focus 按钮 | 点击 `✦ Focus` | 进入 AI Focus 模式 |
| 选中文字 | 选中后自动出现（300ms） | Inline AI 弹框 |

## 四、文档 Tab 管理（§29.3）

点击标题栏中央「当前文档名 ▾」弹出文档切换列表：

| **属性** | **值** |
| --- | --- |
| 列表宽度 | 280px（固定） |
| 最大行数 | 8 条（超出内部滚动） |
| 行高 | 40px |
| 背景 | `--overlay`（`#161616`），`--shadow-lg`，圆角 `--radius-md`（8px） |
| 当前激活文档 | 左侧 ✓ 标记（`#7AA2F7`），背景 `--accent-subtle`，文字 `#F0F0F0` |
| 未保存文档 | 标题右侧显示 `•`（`#7AA2F7`，8px 实心圆） |

**多文档状态管理：** 每个 Tab 独立 AI 对话历史、Context Panel 状态、滚动位置。最多同时打开 10 个文档。快捷键 `Ctrl+Tab` / `Ctrl+Shift+Tab` 切换。

## 五、Context Panel 详细交互（§23.3）

**宽 240px，背景纯黑 `#000000`。** 只做文件树（大纲已独立为编辑器内 Minimap）。

### 文件节点规范

| **状态** | **视觉表现** | **触发条件** |
| --- | --- | --- |
| Default | 背景透明，文字 `#737373`（Inter 13px） | 未选中 |
| Hover | 背景 `rgba(255,255,255,0.04)`，文字 `#B0B0B0` | 鼠标悬停 |
| Active（当前打开） | 左侧 2px `--accent` 指示线 + 背景 `--accent-subtle` + 文字 `#F0F0F0` | 文档在编辑器中打开 |
| Dirty（有未保存修改） | 文件名右侧显示 3px `--brand-warning`（`#F59E0B`）圆点 | `isDirty = true` |

**底部状态栏**：高 36px，显示总字数和连续创作天数（JetBrains Mono 11px `#737373`）。

## 六、编辑器内大纲 Minimap（§23.4 + §29.4）

**大纲贴着编辑器左侧，像一根细竖线，鼠标扫过时才展开为 160px 可操作面板。**

| **状态** | **宽度** | **视觉表现** |
| --- | --- | --- |
| 收起（默认） | 20px | 1px 竖向实线 + 标题位置标记小方块 + accent 色视口指示器 |
| 展开（hover） | 160px | 半透明磨砂背景，标题列表可见，当前标题高亮 |
| 固定展开 | 160px | 常驻，编辑区左移让位，Pin 图标高亮 |
| Hidden | 0 | Zen Mode / AI Focus Mode 时完全隐藏 |

**标题层级：** H1 → Inter 12px/500 `#B0B0B0`，H2 → Inter 11px/400 `#737373`（缩进 10px），H3 → Inter 10px/400 `#555555`（缩进 20px）。

## 七、Toast 通知体系（§23.5 + §29.5）

**设计原则：Toast 是最后手段，不是默认反馈方式。**

### Toast 类型

| **类型** | **图标** | **左侧标识线** | **自动消失** |
| --- | --- | --- | --- |
| 成功 | `CheckCircle` `#22C55E` | `#22C55E` | 3s |
| 错误 | `XCircle` `#EF4444` | `#EF4444` | 8s 或手动 |
| 警告 | `AlertTriangle` `#F59E0B` | `#F59E0B` | 5s |
| 信息 | `Info` `#7AA2F7` | `#7AA2F7` | 4s |
| AI 任务 | `Bot` `#7AA2F7` | `--accent` | 5s |

### 堆叠规则

<!-- 已统一为 §29.5 修订值 -->
- **位置**：右下角，距屏幕边距 16px，`z-index: --z-toast`（400）
- 最多同时显示 3 条，新 Toast 从底部推入
- 鼠标悬停时暂停自动消失计时
- 背景 `#222222`，圆角 8px

### 场景分类（不该弹 Toast 的情况）

- ❌ 常规自动保存——状态栏静默更新
- ❌ 手动 `Ctrl+S` 保存——状态栏「已保存」即可
- ❌ 剪贴板复制——按钮状态变化（图标变 ✓）
- ❌ AI 在当前文档生成完成——结果直接在 AI 面板呈现
- ❌ 轻操作成功（重命名、移动）——Context Panel 直接更新

## 八、第一次启动体验（§28）

### 启动序列（全程 < 8 秒）

| **阶段** | **时长** | **画面** |
| --- | --- | --- |
| ① Splash | 0 – 1.5s | 纯黑背景，✦ 符号 fade in（800ms） |
| ② 品牌展示 | 1.5 – 2.5s | 「CreoNow」字样淡入（Playfair Display 20px `#B0B0B0`） |
| ③A 首次 → 欢迎页 | 2.5s 起 | 见下方欢迎页设计 |
| ③B 再次 → 直接进入 | 2.5s 起 | 直接进入 Dashboard 或上次打开的文档 |

### 欢迎页设计（仅首次启动）

- **只有一个必填项**：工作区名称（可跳过）
- **不要邮箱、不要密码、不要功能选择**
- **背景**：`#050505` 纯黑
- **主标题**：Playfair Display 24px，`#B0B0B0`
- **主按钮**：filled `#7AA2F7`，160px 宽
- 元素 stagger 入场，每项间隔 80ms

### 从不打扰原则（No Interrupt Policy）

- ❌ 不弹出「新功能介绍」浮层
- ❌ 不显示「你知道吗？」Tooltip
- ❌ 不在写作途中弹出任何主动引导
- ✅ 新功能通过状态栏极小圆点静默提示

## 九、App Shell 进场动画（§29.6）

### 进场时序（总时长 ≈ 500ms）

```
t=0ms    Splash fade-out 开始（300ms）
t=100ms  Icon Rail translateX(-48px)→0 + opacity（200ms，--ease-out）
t=180ms  Context Panel translateX(-240px)→0 + opacity（250ms，--ease-out）
t=240ms  标题栏 translateY(-32px)→0 + opacity（200ms，--ease-out）
t=300ms  编辑区内容 opacity 0→1（200ms，--ease-out）
t=420ms  AI FAB scale(0)→1 + opacity（150ms，--ease-spring）
t=500ms  进场完成
```

**再次启动**：stagger 间隔从 80ms 压缩到 40ms，duration 为首次的 70%。

## 十一、多文档分栏对比（§23.8 + §29.8）

| **属性** | **规范** |
| --- | --- |
| 栏数上限 | 最多 3 栏，每栏最小 320px |
| 分隔线 | 1px `rgba(255,255,255,0.08)`，hover 高亮 `--accent-muted` |
| 栏头部 | 高 32px，背景 `#0A0A0A`，文件名 Inter 12px `#737373` |
| 滚动同步 | 默认独立，可开启「滚动同步」模式 |
| AI 跨文档 | AI 上下文范围可选「全部已开栏文档」 |

**触发方式**：`Ctrl+\`、Context Panel 右键「在分栏中打开」、搜索结果中选择。
