# 40.1 构建路线图（Build Roadmap）

> **来源**：`docs/前端决策.md` §13

---

## Round 1：Shell + 文件树 + 静态编辑器

**目标：** 空壳跑通，验证 5 层架构可行性

- [ ] 实现 IPC Bridge Layer（`_bridge.ts` + `service-factory.ts`）
- [ ] 实现 5 个 Domain Service Stub（document / project / stats / export / settings）
- [ ] 实现 Zustand store（Document / UI / Navigation / Settings — 只含骨架 + persist）
- [ ] 实现 Design System Primitives（Button / Input / Badge / Separator / ScrollArea / Tooltip / Switch / Slider / Toggle / Popover）
- [ ] 实现 Shell（AppLayout + IconRail + ContextPanel + 空 RouterOutlet）
- [ ] 实现 EditorFeature（TipTap 基础编辑器 + Source Serif 4 字体加载）
- [ ] 实现 FileTreeFeature（文件树展示 + 新建/重命名/删除/文件夹操作）
- [ ] 实现 AppErrorBoundary + EditorErrorBoundary

**Round 1 DoD：**

- Shell 冷启动 ≤ 800ms
- 编辑器可输入纯文本，Source Serif 4 正确渲染
- 文件树可展开/折叠/新建文件/重命名
- 所有 Primitives 有 Storybook Story（10/10）
- 单元测试覆盖率 ≥ 80%（Service Layer ≥ 95%）

## Round 2：Context Panel + Dashboard + Settings

**目标：** 主要页面可用，验证路由 + 状态管理

- [ ] 实现 ContextPanelFeature 完整交互
- [ ] 实现 DashboardFeature（Heatmap + KPI + Insights）
- [ ] 实现 SettingsFeature（5 个 Tab 各 Section + persist + 实时预览）
- [ ] 实现 Composites（Modal/Dialog + KPICard + HeatmapGrid + SettingsSection + FileNode + FolderNode）
- [ ] 实现 NavigationController（路由动画 + 面板状态）
- [ ] 实现 SidebarErrorBoundary + PanelErrorBoundary

**Round 2 DoD：**

- Dashboard Heatmap 数据加载 ≤ 200ms（IPC → render）
- Settings 修改立刻预览且持久化（关闭窗口 → 重开不丢失）
- 路由动画 ≤ 150ms
- 所有 Composites 有 Storybook Story（12/12）
- E2E：编辑 → 保存 → Dashboard 显示当天 heatmap 色块更新（≥ 3 场景）

---

## Round 3：AI 面板 + 流式 + FAB

**目标：** 选中文本 → 调用 AI 技能 → 流式输出 → 接受/拒绝

- [ ] 实现 `ai.service.ts` + Stream Manager
- [ ] 实现 `useAiStream` hook（rAF 批量更新）
- [ ] 实现 AiPanelFeature（Cursor 式对话面板 + Agent/Plan/Analyze/Ask 模式 + 底部 Skill 弹出框）
- [ ] 实现 AI FAB（浮动操作按钮：拖拽/单击/双击/长按触发 Focus Mode）
- [ ] 实现 AI 内联辅助 UI（ghost text + 流式渲染）
- [ ] 实现 AI 回滚（EditorState 快照恢复）
- [ ] 实现背压配合（降级 UI）
- [ ] 实现 PanelErrorBoundary

**Round 3 DoD：**

- AI 流式输出帧率 ≥ 55 FPS（rAF 批量期间）
- 首 token 上屏延迟 ≤ 100ms
- 回滚恢复 ≤ 50ms，EditorState 完全恢复到生成前
- 背压触发后 UI 降级显示骨架动画（无卡死）
- E2E：选中文本 → 选技能 → 流式生成 → 接受/拒绝（≥ 5 场景）

## Round 4：知识图谱 + 记忆

**目标：** KG 实体管理 + 实体识别 + 记忆注入可视化

- [ ] 实现 `knowledge.service.ts` + `memory.service.ts`
- [ ] 实现 KnowledgeGraphFeature（实体/关系 CRUD + 可视化）
- [ ] 实现 `entityMention` 扩展（@ 引用实体）
- [ ] 实现 `entityHighlight` 扩展（自动识别高亮）
- [ ] 实现 MemoryFeature（记忆列表 + 蒸馏进度 + 注入预览）

**Round 4 DoD：**

- KG 实体 CRUD 全路径 E2E 通过
- @ 实体搜索响应 ≤ 200ms
- 实体高亮不影响编辑帧率（≥ 55 FPS）
- 记忆蒸馏进度 UI 实时更新（轮询间隔 ≤ 2s）
- E2E：创建角色 → @ 引用 → 高亮 → hover 卡片（≥ 5 场景）

- [ ] 实现 VersionFeature（版本历史 + diff + 分支 + 合并）
- [ ] 实现 SearchFeature（全局搜索 + 搜索替换）
- [ ] 实现 ExportFeature（MD/TXT/DOCX/PDF/Bundle）

**Round 5 DoD：**

- 导出格式 5 种全部可用
- 写作统计数据实时更新（延迟 ≤ 1s）
- 10 个 Feature 全部就位；E2E 场景总计 ≥ 50 个
