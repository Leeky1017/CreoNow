# Editor Specification Delta

## Change: s2-write-button

### Requirement: 编辑器必须提供续写悬浮按钮并绑定写作技能调用 [ADDED]

系统必须在编辑器中提供续写悬浮按钮组，用于在不离开当前编辑上下文的前提下触发写作技能。

- 当满足显示条件时，按钮组必须可见并可交互。
- 按钮触发必须绑定到稳定的写作技能标识（来自 `s2-writing-skills`）。
- 交互测试必须覆盖可见性与调用参数，避免仅渲染通过的假测试。

#### Scenario: 满足条件时显示续写按钮组 [ADDED]

- **假设** 编辑器处于可编辑状态且光标位于可续写位置
- **当** 用户停留在目标位置并触发悬浮交互
- **则** 系统显示续写按钮组
- **并且** 按钮状态与当前编辑器上下文一致（可点/禁用）

#### Scenario: 点击续写按钮触发 write 技能调用 [ADDED]

- **假设** 续写按钮组已显示且写作技能清单可用
- **当** 用户点击续写按钮
- **则** 系统发起写作技能调用，技能标识为 `write`
- **并且** 传入调用参数包含当前编辑上下文所需最小字段

## Out of Scope

- Bubble Menu AI 按钮与选中文本子菜单交互
- 快捷键与 Slash 命令入口
- SkillExecutor 运行时策略调整
