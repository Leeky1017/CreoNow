# Editor Specification Delta

## Change: s2-slash-commands

### Requirement: Slash 写作命令注册与执行路由 [ADDED]

系统必须提供 Slash 命令注册表，并注册 Sprint 2 路线图定义的 6 个写作命令：`/续写`、`/描写`、`/对白`、`/角色`、`/大纲`、`/搜索`。

- 每条命令必须有稳定命令 id 与展示文案。
- 命令列表必须支持按用户输入关键词过滤。
- 用户确认命令后必须按命令 id 路由到对应执行入口。

#### Scenario: 命令注册表完整加载 6 个命令 [ADDED]

- **假设** Slash 面板请求命令源
- **当** 系统初始化命令注册表
- **则** 返回包含 `/续写`、`/描写`、`/对白`、`/角色`、`/大纲`、`/搜索` 的命令集合
- **并且** 每条命令都具备唯一命令 id

#### Scenario: 输入关键字后过滤命令列表 [ADDED]

- **假设** 命令注册表已加载
- **当** 用户输入关键字用于筛选
- **则** 返回仅包含匹配关键字的命令子集
- **并且** 清空关键字后恢复全量命令列表

#### Scenario: 选择命令后触发对应执行入口 [ADDED]

- **假设** 用户在面板中定位到目标命令
- **当** 用户确认该命令
- **则** 系统按该命令 id 调用对应执行器
- **并且** 不触发非目标命令的执行逻辑
