# Editor Specification Delta

## Change: s2-slash-framework

### Requirement: Slash Command 扩展框架与命令面板 [ADDED]

系统必须提供 TipTap Slash Command 扩展框架：在编辑态输入 `/` 时展示命令面板，并支持基础搜索过滤与关闭操作。

- Slash 扩展负责监听输入触发并维护面板打开状态。
- 命令面板必须包含搜索输入区与候选命令列表区。
- 面板关闭后不得干扰普通文本编辑流程。

#### Scenario: 输入斜杠触发命令面板 [ADDED]

- **假设** 用户处于可编辑文档且光标位于正文
- **当** 用户输入 `/`
- **则** 命令面板出现并展示默认候选列表
- **并且** 编辑器保持当前选区上下文

#### Scenario: 搜索关键字过滤候选命令 [ADDED]

- **假设** 命令面板已打开且存在多条候选命令
- **当** 用户在面板搜索框输入关键字
- **则** 列表仅保留匹配项并实时更新
- **并且** 不匹配时显示空结果态

#### Scenario: 关闭面板后恢复普通输入 [ADDED]

- **假设** 命令面板处于打开状态
- **当** 用户执行关闭操作（例如按 `Escape`）
- **则** 命令面板被收起
- **并且** 后续输入按普通文本编辑处理
