# Knowledge Graph Specification Delta

## Change: s2-entity-matcher

### Requirement: 实体匹配引擎必须支持名字与别名匹配并满足性能阈值 [ADDED]

系统必须提供可验证的实体匹配能力，覆盖实体名与别名输入，并满足路线图定义的性能门槛。

#### Scenario: KG-S2-EM-S1 名字与别名匹配可验证 [ADDED]

- **假设** 实体集合包含 `name` 与 `aliases`
- **当** 输入文本包含实体名或任一别名
- **则** 匹配结果返回对应实体
- **并且** 同一实体的重复命中按既定规则去重

#### Scenario: KG-S2-EM-S2 性能阈值可验证 [ADDED]

- **假设** 基准数据规模为 100 个实体、1000 字输入文本
- **当** 执行实体匹配
- **则** 单次匹配耗时小于 10ms
- **并且** 性能断言由可重复执行的测试固定

#### Scenario: KG-S2-EM-S3 边界输入可验证 [ADDED]

- **假设** 输入覆盖重叠名称、中文文本和空输入
- **当** 执行实体匹配
- **则** 输出结果保持确定性且无异常
- **并且** 空输入返回空匹配结果
