# 黄金标准参考实现

> **用途**: 提供代码级别的质量基准  
> **来源**: 经过精雕细琢和验收的组件实现

---

## 什么是黄金标准

黄金标准组件是：

1. **质量标杆**: 代表最佳实践的代码实现
2. **风格基准**: 后续组件必须参考的代码风格
3. **模式示范**: 展示状态处理、边界处理的正确方式

---

## 黄金标准组件列表

| 组件   | 目录      | 状态   | 说明         |
| ------ | --------- | ------ | ------------ |
| Button | `Button/` | 待创建 | 交互组件模式 |
| Input  | `Input/`  | 待创建 | 表单组件模式 |
| Card   | `Card/`   | 待创建 | 容器组件模式 |

---

## 目录结构

每个黄金标准组件目录包含：

```
Button/
├── Button.tsx          # 组件实现
├── Button.types.ts     # 类型定义
├── Button.stories.tsx  # Storybook Story
├── Button.test.tsx     # 测试用例
├── index.ts            # 导出
└── NOTES.md            # 实现笔记（可选）
```

---

## 使用方式

生成新组件时：

1. **先读取黄金标准代码**
2. **按照相同的代码结构组织**
3. **按照相同的命名约定命名**
4. **按照相同的状态处理方式处理状态**
5. **按照相同的测试模式编写测试**

---

## 创建黄金标准的流程

```
1. AI 根据组件卡片生成初版代码
2. AI 用 Storybook + MCP 自检
3. 用户在 Storybook 中查看，给出反馈
4. AI 根据反馈修改
5. 循环直到用户完全满意
6. 代码归档到 reference-implementations/
7. 该组件成为后续组件的参考模板
```

---

## 代码风格要求

黄金标准代码必须：

- [ ] 使用 TypeScript 严格模式，无 any
- [ ] 所有颜色使用 CSS Variables
- [ ] 使用 Tailwind + cn() 组合样式
- [ ] 完整处理所有状态（见组件卡片）
- [ ] 完整处理所有边界情况
- [ ] 有完整的 JSDoc 注释
- [ ] 有完整的 Storybook Story
- [ ] 有完整的测试用例

---

## 注意事项

- 黄金标准组件一旦确定，**轻易不要修改**
- 如需修改，必须同步更新依赖它的其他组件
- 每次修改必须重新进行完整验收
